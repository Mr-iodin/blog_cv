    1. 查找的基本概念⚓︎

    ​ 定义：给定一个值k，在含有n个记录的表中找出关键字等于k的记录。

    定义一个记录的结构体类型如下：

    1
2
3
4
5
    typedef struct
{
  int key; //关键字的域（有点类似于数据库里的主键）
  ... //其他域
}Elemtype;

    采用何种方法进行查找的相关因素如下：

    • 使用何种数据结构来表示查找表；
    • 查找表中的关键字次序（即无序集合的查找还是有序集合的查找）；

    查找算法的基本操作是关键字的比较，并且关键字的比较次数与待查找关键字有关，因此会把查找过程中对关键字的平均比较次数（也称为平均查找长度）作为衡量一个查找算法效率优劣的标准。平均 查找长度用ASL来表示： $$ ASL=\sum^{n}{i=1} p\times c_{i} $$ ​ 其中，n表示查找表中记录的个数；pi是查找第i个记录的概率（考研题目中一般取\(\frac{1}{n} \(）；\)c_i\)是找到第i个记录所需要进行比较的次数，即查找长度。

    2.基于线性表的查找算法⚓︎

    2.1 顺序查找法⚓︎

    基本思路：从表的一端开始，顺序扫描线性表，依次将扫描到的关键字与给定值k进行比较，若当前扫描的关键字与k相等，则查找成功；扫描结束后仍未发现关键字等于k的记录，则扫描失败。

    • 对于顺序表，可以通过数组下标递增顺序扫描各个元素；
    • 对于链表，可以通过结点指针依次扫描。

    【例】数组a[]中有n个整数，没有次序，数组从下标1开始存储，请写出查找任一元素k的算法，若查找成功，返回元素在数组中的位置，若不成功，则返回0。

    1
2
3
4
5

7
    int Search(int a[],int n,int k)
{
  for(int i;i<=n;i++)
    if(a[i]==k)
      return 1;
  return 0;
}

    ​ 根据k的取值不同，体现了两种查找长度：查找成功的查找长度、查找失败的查找长度。所以ASL也有两种：

    • 查找成功情况下的\(ASL_1\);
    • 查找失败情况下的\(ASL_2\);

    ​ 对于第一种：\(p_i=1/n\),\(c_i=i\),因为若k等于a[i]，则在扫描到的a[i]之前已经进行了i-1次比较，加上最后一次，一共进行了i次比较，因此有 $$ ASL_{1}=\sum^{n}_{i=1} \frac{i}{n} =\left( 1/n\right) \times n\times \left( 1+n\right) /2=\left( n+1\right) /2 $$ ​ 对于第二种：k在a[]中的值之外的范围内取值，这时候k的取值是无限的，但对于k的任意一个取值，其长度必为n。因此\(ASL_2=n\)。

    ​ 由\(ASL_1\)的表达式可以求出时间复杂度为O(n)，由\(ASL_2\)表达式同样求得时间复杂度为O(n)。

    int Bsearch(int R[],int low,int high,int k)
{
  int mid;
  while(low<=high)
  {
    mid=(low+high)/2;
    if(R[mid]==k)
      return mid;
    else if(R[mid]>k)
      high=mid-1;
    else
      low=mid+1;
  }
  return 0;
}

    折半查找法可用二叉树来表示，得到的二叉树称为描述折半查找的判定树。

    2.3 分块查找⚓︎
    • 数据结构

    ​ 分块查找又称为顺序查找，基本原理为：把线性表分为若干块。每一块中的元素存储顺序是任意的，但是块与块之间必须按照关键字大小有序排列（即前一块中最大的关键字要小于下一块中最小的关键字）。

    ​ 对顺序表的分块查找需要额外建立一个小的索引表，表中每一项对应线性表中的一块，每个索引项都由健值分量组成，健值分量存放对应块的最大关键字，链值分量存放指向本块第一个元素和最后一个元素的指针。

    索引表定义如下：

    typedef struct
{
  int key;  //假设表内元素为int型
  int hight,low;    //记录某块中第一个和最后一个元素
}indexElem;
indexElem index[maxSize];   //定义索引表

    • 算法描述

    ​ 可以分为两步进行，首先确定待查找的元素属于哪一块，然后在块内精确查找该元素。由于索引表是递增有序的，因此第一步采用二分查找，块内元素一般个数较少，所以第二步采用顺序查找即可。

    ​ 分块查找实际上进行两次查找，整个算法的平均查找长度是两次查找的平均查找长度之和，即二分查找的平均查找长度+顺序查找的平均查找长度。

    3.基于树的查找算法⚓︎

    3.1 二叉排序树定义⚓︎
    • 定义

    ​ 二叉排序树（BST）或者是空树，或者是满足以下性质的二叉树：

    • 若它的左子树不空，则左子树上所有关键字的值均不大于（不小于）根关键字的值；
    • 若它的右子树不空，则右子树上所有关键字的值均不小于（不大于）根关键字的值；
    • 左右子树又各是一棵二叉排序树。

    ​ 若要输出该二叉排序树的有序序列，则只需要输出二叉排序树的中序遍历序列。

    • 存储结构

    通常采用二叉链表进行存储。

    BTNode* BSTSearch(BTNode* bt,int key)
{
  if(bt==NULL)
    return NULL; //结点为空，查找不成功返回NULL
  else
  {
    if(bt->key==key)
      return bt;    //结点为关键字，查找成功返回结点指针
    else if(key<bt->key)
      return BSTSearch(bt->rchild,key);
    else 
      return BSTSearch(bt->lchild,key);
  }
  //*最后两个return作用树逐层将内部函数返回值传给上层，最终由外层函数返回，来告诉用户执行是否成功。
}

    • 删除关键字操作

    ​ 在二叉树中删除一个关键字时，不能把该关键字所在的结点为根的子树都删除，而是只删除这一结点，并保持二叉树的特性。

    ​ 假设二叉排序树上被删除的结点为p，f为其双亲结点，则删除结点p的过程分为以下三种：

    • p结点为叶子结点。直接删除即可；
    • p结点只有右子树而无左子树，或者只有左子树无右子树。此时将p删除，然后将p的子树直接连接到原来p于双亲结点f相连的指针上即可。
    • p结点既有左子树又有右子树。先沿着p的左子树根结点的右指针一直往右走，直到来到其右子树最右边的一个结点r（也可以沿着p的右子树根结的左指针一直往左走，直到来到其左子树的最左一个结点）。然后将p中的关键字用r中的关键字来替代。最后判断，如果r是叶子结点，直接删除，如果r是非叶子结点，则按照第二个方法删除r。

    3.3 平衡二叉树⚓︎
    • 概念

    ​ 平衡二叉树又称AVL，是一种特殊的二叉排序树。其左右子树都是平衡二叉树，且左右子树高度之差绝对不超过1。以树中所有结点为根的树的左右子树高度之差绝对值不超过1。

    为了判断一棵树是否为平衡二叉树，引入了平衡因子的概念。平衡因子是针对树中的结点来说的，一个结点的平衡因子为其左子树的高度减去右子树高度的差。

    对于平衡二叉树来说，一个结点的平衡因子只能是-1、0、1。

    3.4 平衡二叉树的基本算法（待补充）⚓︎

    • 平衡二叉树的建立

      建立平衡二叉树的过程和建立二叉排序树的过程一样，都是将关键字逐个插入空树中。不同的是，平衡二叉树对每插入的一个新的关键字都要进行检查，看是否新关键字的插入会使得原平衡二叉树会失去平衡。如果失去平衡则需要进行平衡调整。

    • 平衡调整

    假定向平衡二叉树中插入一个新结点后破坏了原平衡二叉树的平衡，则首先要找到插入后失去平衡的最小子树，然后调整这颗子树，使其成为平衡子树。

    当失去平衡的最小子树被调整为平衡子树后，无需调整原有其他所有不平衡子树，整个二叉排序树就会成为一棵平衡二叉树。

    当然平衡调整必须保持排序二叉树左小右大的性质，平衡调整有4种情况，分别为LL型，RR型，LR型，RL型。

    4.B-树与B+树⚓︎

    4.1 B-树（B树）的基本概念⚓︎

    B-树中所有结点的孩子结点个数最大值称为B-树的阶，通常用m表示，从查找效率来说，要求m ≥ 3。一棵m阶的B-树或者是一棵空树，或者是满足以下要求的m叉树。

    • 每个结点最多有m个分支（子树）；而最少分支数要看是否为根结点，如果是根结点且不是叶子结点，则至少有两个分支，非根非叶结点至少有$\left\lceil m/2\right\rceil $个分支；

    • 有n（k≤n≤m）个分支的结点有n-1个关键字，它们按照递增顺序排列。k=2（根结点）或\(\left\lceil m/2\right\rceil\)（非根结点）。

    • 每个结点的结构为

    ​ 其中，n为该结点中关键字的个数；\(k_i\)(1≤i≤n)为该结点的关键字且满足\(k_i\)<\(k_{i+1}\);\(p_i\)(0≤i≤n)为该结点的孩子结点指针且满足(0≤i≤n-1)所指结点上的关键字大于\(k_i\)且小于\(k_{i+1}\),\(p_0\)所指结点上的关键字小于\(k_1\),\(p_n\)所指结点上的关键字小于\(k_n\)。

    • 结点内各关键字互不相等且按从小到大排列。

    • 叶结点处于同一层；可以用空指针表示，是查找失败到达的位置。

    上图为一棵5阶的B-树，高度为3，总结点树为30：

    ​ 1）结点分支等于关键字数+1，最大的分支数就是B-树的阶数，因此m阶B-树最多有m个分支。

    ​ 2）图为一棵5阶B-树，所以\(\left\lceil m/2\right\rceil =\left\lceil 5/2\right\rceil=3\)，可以看出，图中除了根结点外，所有结点至少有3个分支。根结点可以不满足这个条件，图中根有两个分支。

    ​ 3）如果根结点中没有关键字就没有分支，此时B-树是一棵空树；如果根结点有关键字，则其分支树必大于2，因为分支数等于关键字数加1。

    ​ 4）除根结点外，结点中关键字的个数至少为2，因为分支数至少为3，分支数比关键字数多1; 还可以看出结点内关键字都是有序的，并且在同一层上，左边结点内所有关键字均小于右边结点内所有关键字。

    ​ B-树中一个很重要特征，下层结点内的关键字取值总是落在由上层结点关键字所划分的区间内，具体落在哪个区间内可由指向它的指针看出。

    ​ 5）B-树叶子结点均在第4层上，代表查找不成功的位置

    4.2 B-树的基本操作⚓︎
    • B-树关键字查找

    ​ 查找方法是二叉树的扩展，二叉排序树是二路查找，B-树是多路查找。查找步骤具体如下：

    • 先让key与根结点中的关键字进行比较，若key==k[i],则查找成功；
    • 若key<k[1]，则到p[0]所指示的子树中进行继续查找。
    • 若key>k[n]，则到p[n]所指示的子树中继续查找。
    • 若key[i]<key<k[i+1]，则沿着指针p[i]所指示的子树中继续查找。
    • 若遇到空指针，则查找不成功。

    ​ B-树查找关键字42。首先看根结点42>30，沿着根结点中p[1]往下走，在子树根结点中查找，39<42<45，则沿着树根结点中指针p[1]往下走，在下层结点中查找关键字42成功，查找结束。

    • B-树关键字的插入

    对于关键字的插入，需要找到插入位置。在B-树查找过程中，当遇到空指针时，则证明查找不成功，同时找到了插入位置，即根据空指针可以确定在最底层非叶结点（终端结点）中插入位置。B-树新关键字的插入总是落在终端结点上。

    4.3 B+树的概念⚓︎

    ​ B+树是B-树的一种变形，它与B-树有很多相似之处。这里可以对比记忆：

    • 在B+树中，具有n个关键字结点含有n个分支；而在B-树中，具有n个关键字结点含有n+1个分支。
    • 在B+树中，每个结点（除根结点之外）中的关键字个数n的取值范围为\(\left\lceil m/2\right\rceil \leq n\leqslant m\)，根结点的取值范围为\(2\leq n\leqslant m\)；而在B-树中，它们的取值范围分别是\(\left\lceil m/2\right\rceil-1 \leq n\leqslant m-1\)和\(1\leq n\leqslant m-1\)​。
    • 在B+树中叶子结点包含信息。并且包含全部关键字，叶子结点引出的指针指向记录。
    • B+树中所有叶子结点仅起到一个索引的作用，即结点中的每个索引项只含有对应子树的最大关键字和指向该子树的指针，不含有该关键字对应记录的存储地址；而在B-树中，每个关键字对应一个记录的存储地址。
    • 在B+树上有一个指针指向关键字最小的叶子结点，所有叶子结点链接成一个线性链表，而在B-树中没有。

    5.散列表⚓︎

    5.1 散列表的概念⚓︎

    ​ 根据给定的关键字来计算出关键字在表中的地址。

    ​ 在查找表中，关键字的地址跟关键字之间不存在确定的关系；而在Hash表中，关键字和关键字的地址具有确定关系。这种关系可以用Hash函数的H来表示。

    例如：关键字为key，则H(key)称为Hash地址，就是key在查找表中的地址。

    5.4 ASL1与ASL2⚓︎

    ​ 在等概率情况下查找成功和不成功时的平均查找长度ASL1和ASL2。

    • 要求ASL1，关键是求出对于查找每个关键字所对应的比较次数。通过上面分析可知，如果没有冲突，则需比较一次；若发生冲突，就根据其冲突解决方法计算出比较次数。

    根据上表可知： $$ ASL_1=(1+2+1+2+2+1+2+1+2+8)/10=11/5 $$

    • 要求ASL2，关键是求出查找不成功的情况下比较次数。这里比较次数是针对表中每个可以通过Hash函数计算得到的地址来说的，对于每个地址求出由这个地址开始的比较次数，然后求其平均数就是查找不成功的平均查找长度。

    由上表知： $$ ASL_2=(1+3+2+1+9+8+7+6+5+4+3+2+1)/13=4 $$

    5.5 其他构造方法和冲突解决方法⚓︎

    • 常用的几种构造方法

    • 直接定址法

      ​ 取关键字或关键字的某个线性函数为Hash地址，即H(key)=key或者H(key)=a×key+b,其中a和b为常数。

    • 数字分析法

      ​ 假设关键字是r进制数（如十进制），并且Hash表中可能出现的关键字都是事先知道的，则取关键字若干数位组成Hash地址。

    • 平方取中法

      ​ 取关键字平方后的中间几位作为Hash地址。通常选定Hash函数时不一定能知道关键字的全部情况，仅取其中几位地址为地址不一定合适，而一个数平方后的中间几位数和数的每一位都相关，由此得到Hash地址随机性更大。

    • 除留余数法

      ​ 取关键字被某个不大于Hash表表长m的数p除后所得的余数为Hash地址，即 $$ H(key)=key\quad mod\quad p\quad(p\leqslant m) $$ 在本方法中，p的选择很重要，一般p选择小于或者等于表长的最大素数，这样可以减少冲突。

    （以上4种方法，除留余数在考研中涉及最多，考生要结合上例熟练掌握）

    • 常用Hash冲突处理方法

    • 开放定址法

      ​ 在开放定址法中，以发生冲突的Hash地址为自变量，通过某种冲突解决函数得到一个新的空闲Hash地址的方法。

      ​ 1）线性定址探查法（即例题中所用的方法）

      ​ 线性探查法容易产生堆积问题。

      ​ 2）平方探查法。

      ​ 设发生冲突的地址为d，则用平方探查法所得到的新地址序列为\(d+1^2\),\(d-1^2\),\(d+2^2\),\(d-2^2\)​...平方探查法是一种较好的处理冲突的方法，可以减少堆积问题。缺点是不能探查到Hash表上的所有单元，但至少能探查到一半单元。

    • 链地址法

      链地址法是把所有的同义词用到单链表链接起来的方法。在这种方法中，Hash表每个单元中存放的不再是记录本身，而是相应同义词单链表的表头指针。
